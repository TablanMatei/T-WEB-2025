Specificație Cerințe Sistem - BoW (Books on Web)

1. Introducere

1.1 Scopul aplicației

BoW este o aplicație web destinată utilizatorilor individuali sau grupurilor (de exemplu, clase școlare sau cluburi de lectură) pentru a gestiona progresul lecturilor și a exprima opinii sau adnotări referitoare la cărțile citite.

1.2 Domeniul aplicației

Aplicația oferă management complet al cărților parcurse, organizarea acestora pe diverse criterii, generarea de statistici exportabile, recomandarea bibliotecilor publice din proximitate și fluxuri de știri RSS legate de lecturi și noutăți.

2. Descriere generală

2.1 Perspective sistem

BoW este o platformă web care interacționează cu utilizatorii printr-o interfață intuitivă și responsivă. Integrarea cu API-uri externe, cum ar fi Google Books API și Open Library API, oferă date suplimentare despre cărți.

2.2 Funcționalități principale

  • Înregistrarea progresului în lectură.
  • Exprimarea opiniilor, recenziilor personale.
  • Organizarea cărților pe criterii precum categorie, autor(i), editură, an, ediție.
  • Generarea și exportul statisticilor în formate deschise: CSV, JSON, XML
  • Recomandarea bibliotecilor publice din proximitate (via Overpass API by OpenStreetMap).
  • Flux RSS cu anunțuri legate de lecturi și noutăți din domeniul literar.
  • Bonus: integrare cu Google Books API pentru informații suplimentare.

3. Cerințe detaliate

  • RF1: Sistemul trebuie să permită utilizatorilor să marcheze progresul lecturii pentru fiecare carte.
  • RF2: Utilizatorii pot adăuga opinii și adnotări pentru cărțile citite.
  • RF3: Cărțile pot fi organizate după autor, categorie, editură, an și ediție.
  • RF4: Sistemul trebuie să sugereze biblioteci publice din proximitate dacă o carte nu este disponibilă.
  • RF5: Aplicația trebuie să genereze statistici exportabile în format minimal, CSV și DocBook.
  • RF6: Sistemul oferă un flux RSS pentru anunțuri relevante privind lecturile.
  • RF7 (Bonus): Integrarea cu Google Books API și Open Library API pentru date suplimentare despre cărți.

4. Interacțiunea cu utilizatorul

Utilizatorii accesează platforma prin browser web modern, având posibilitatea să vadă și să modifice progresul lecturilor, să adauge recenzii și să vizualizeze recomandări personalizate.

Navigarea este intuitivă, cu meniuri clare, filtre pentru categorii și opțiuni de sortare, facilitând găsirea rapidă a cărților de interes.

Notificările și fluxul RSS oferă o modalitate simplă de a rămâne la curent cu noutățile și anunțurile importante legate de lecturi.

5. Proiectarea și implementarea interfeței Web responsive

5.1 Utilizarea specificațiilor HTML și CSS moderne

Interfața Web a fost proiectată și implementată folosind cele mai recente specificații HTML5 și CSS3, asigurând compatibilitatea cu browserele moderne și o experiență de utilizare optimă pe diverse dispozitive. Structura semantică a paginilor HTML facilitează accesibilitatea și întreținerea codului, iar stilizarea CSS utilizează tehnici precum Flexbox și Grid pentru aranjarea elementelor în pagină.

5.2 Responsivitate

Pentru a asigura o experiență consistentă pe desktop, tabletă și mobil, s-au folosit media queries și unități relative (%, rem, em). Elementele interfeței se adaptează automat la dimensiunea ecranului, meniurile devin accesibile pe mobil, iar conținutul rămâne lizibil și ușor de navigat indiferent de dispozitiv.

5.3 Motivații pentru alegerile de design

Alegerea unui design minimalist și intuitiv a fost motivată de dorința de a oferi utilizatorilor o interacțiune rapidă și fără obstacole. Paleta de culori neutră și contrastantă asigură lizibilitate, iar butoanele și elementele interactive sunt evidențiate clar. S-a pus accent pe simplitate, pentru a reduce curba de învățare și a crește eficiența utilizării aplicației.

S-a optat pentru separarea clară a secțiunilor (ex: genuri, grupuri, cărți finalizate, setări) pentru a facilita navigarea și a reduce confuzia. Fiecare pagină are o structură coerentă, cu titluri vizibile și acțiuni principale ușor accesibile.

5.4 Exemplu de implementare

  • Folosirea Flexbox pentru aranjarea cardurilor de cărți și a listelor de genuri.
  • Media queries pentru adaptarea meniului și a layout-ului pe ecrane mici.
  • Buton de meniu tip „hamburger” pe mobil pentru acces rapid la secțiuni.
  • Fonturi web sigure și iconuri SVG pentru claritate pe orice rezoluție.

Concluzie: Designul responsive și alegerile de stil au fost făcute pentru a maximiza accesibilitatea, ușurința în utilizare și atractivitatea vizuală, respectând în același timp bunele practici actuale din dezvoltarea web.
